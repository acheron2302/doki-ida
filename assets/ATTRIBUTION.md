# Asset Attribution

The character stickers, full-listing wallpapers, and theme color
definitions shipped with this plugin originate from the **doki-theme**
project by Alex Simons (Unthrottled) and contributors, and are reused
here under that project's terms.

- Theme definitions: https://github.com/doki-theme/doki-master-theme
  (`definitions/**/**.master.definition.json`) — pinned to a specific
  commit in `third_party/doki-master-theme/UPSTREAM_PIN.json` and
  refreshed via `tools\fetch_upstream.ps1`.
- Sticker artwork: https://github.com/doki-theme/doki-theme-assets
  (`stickers/vscode/**`)
- Wallpaper artwork: https://github.com/doki-theme/doki-theme-assets
  (`backgrounds/wallpapers/transparent/**`)

The plugin does not bundle image files. The asset manager downloads
the official PNGs on first use from
`https://doki.assets.unthrottled.io` and caches them locally, so the
on-disk binaries come directly from the upstream doki-theme-assets
repository and remain under that project's terms.

Character artwork is the property of the respective anime studios /
original artists; doki-theme distributes it for theming purposes.

### Theme metadata format

The runtime ships a single `theme_catalog.json` generated by
`tools\generate_theme_catalog.ps1`. Each theme entry includes:

- `id`, `name`, `displayName`, `group`, `author`, `dark`, `product`
- `colors` (the master palette) and `overrides.editorScheme.colors`
  (which the runtime merges on top of `colors`).
- `stickers.default` and optional `stickers.secondary`, each with
  `name`, `anchor`, and `remotePath` (the authoritative upstream CDN
  path).
- `background` with `name`, `anchor`, and `remotePath` pointing at
  the transparent variant under
  `backgrounds/wallpapers/transparent/<name>`.

### Worked example paths

The catalog generator derives the upstream CDN path from each
definition's relative path under `definitions/` (the same logic
`doki-build-source.resolveStickerPath` uses in the official
VS Code build). For example:

| Definition path (relative to repo root)                                  | Sticker CDN path                                              |
|--------------------------------------------------------------------------|---------------------------------------------------------------|
| `definitions/reZero/rem/rem.master.definition.json`                     | `stickers/vscode/reZero/rem/rem.png`                          |
| `definitions/franxx/zeroTwo/obsidian/zero.two.obsidian.master.definition.json` | `stickers/vscode/franxx/zeroTwo/obsidian/zero_two_obsidian.png` |
| `definitions/franxx/zeroTwo/sakura/zero.two.sakura.master.definition.json`     | `stickers/vscode/franxx/zeroTwo/sakura/zero_two_sakura.png`   |
| `definitions/konoSuba/darkness/dark/darkness.dark.master.definition.json`      | `stickers/vscode/konoSuba/darkness/dark/darkness_dark.png`    |
| `definitions/konoSuba/darkness/light/darkness.light.master.definition.json`    | `stickers/vscode/konoSuba/darkness/light/darkness_light.png`  |

Every wallpaper path is the transparent variant of the same file
name, e.g. `stickers/vscode/.../rem.png` →
`backgrounds/wallpapers/transparent/rem.png`.
